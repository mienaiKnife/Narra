# Playback Service & Lifecycle

Audio playback in Narra is handled by a dedicated `PlaybackService` to ensure it continues even when the app is in the background or the screen is off.

## Architecture

Narra uses **Media3 / ExoPlayer** for audio management and a **MediaSession** to integrate with the Android system (lock screen controls, Bluetooth, etc.).

### Components
- **`PlaybackService`**: A `MediaSessionService` that lives in the `service` package. It manages the `ExoPlayer` instance.
- **`MediaSession`**: Provides a way for external controllers (like the system UI or Android Auto) to interact with the playback.
- **`TtsAudioSource`**: A custom Media3 `MediaSource` or `DataSource` that feeds the audio chunks generated by the `TtsEngine` into the player.

## The Playback Flow

1. **Queueing**: Articles are added to the `PlaybackQueue`.
2. **Loading**: When an article starts playing, the `PlaybackService` requests audio from the active `TtsEngine`.
3. **Buffering**: The TTS engine synthesizes the text in chunks. These chunks are buffered and fed into the player to minimize latency.
4. **Foreground Status**: When playback starts, the service moves to the foreground and shows a persistent notification. It exits the foreground when paused or stopped.

## Handling Interruptions

### Audio Focus
The service automatically handles audio focus changes (e.g., ducking for notifications, pausing for incoming calls).

### Connection Changes
For cloud-based TTS engines, the service monitors network connectivity and handles playback errors gracefully by pausing or retrying.

## Contributing to Playback
If you are modifying the playback logic:
- **Do not control the service from ViewModels**: ViewModels should only send commands (play, pause, skip) and observe the state via a repository or flow.
- **Respect the Lifecycle**: Ensure the `MediaSession` is released and the player is destroyed in `onDestroy()` to avoid memory leaks.
- **Battery Optimization**: Use appropriate wake locks if necessary, but keep power consumption in mind.
